⚓ Overview

SPICA TIDE is a professional deck cargo planning application built for Platform Supply Vessel SPICA TIDE operating in the North Sea under NEO Energy. It provides an interactive drag-and-drop deck plan with real-time cargo visualisation, platform colour coding, DG safety checks, and full PDF/Excel manifest export.

The application replaces paper-based and spreadsheet-based cargo plans with a precise, scalable digital tool that officers can use at the planning desk or on a bridge tablet.

🚢 12 Cargo Bays
Full deck from Bay 12 (aft) to Bay 1 (bow), each individually labelled and sized to real vessel dimensions.
⚖ 7 Platforms
Bleo Holm, Claymore CAP/CPP/WOPS/Drill, Piper, Saltire — each with unique colour coding and cargo tracking.
⚠ DG Safety
Full IMDG segregation matrix with 20 DG classes. Automatic pairwise conflict detection with visual warnings.
📄 Export Ready
Professional PDF deck plans and Excel manifests exported directly to disk via native save dialogs.

⚙ Core Features

👁 Drag & Drop Placement

Select cargo from the Library panel and drag it directly onto the deck canvas. Drop to place at the exact position. Cargo blocks can be repositioned by dragging, rotated 90° with the R key, duplicated with D, or deleted with Del.

📚 Cargo Library Panel

The bottom panel contains 60+ preset cargo types across 5 categories: Container, Module, Tank, Basket, Skip. Each preset includes real-world dimensions in metres and default weights. Custom cargo types can be added and saved.

🎨 Platform Colour Coding

Each platform destination is assigned a unique colour. Cargo blocks are tinted by their assigned platform, making it easy to visually identify which cargo goes where. Colours can be manually changed by clicking the colour dot on any location card.

📌 Cargo Statuses

Four cargo statuses drive colour tinting and manifest grouping:

■ Load
Cargo being loaded onto vessel for delivery
■ Backload
Cargo being returned from platform to shore
■ ROB
Remaining On Board from previous voyage
■ Transfer
Cargo transferring between platforms (dashed border)

📄 ASCO / iLMS Excel Import

Upload an ASCO loadout Excel file (.xlsx) and the app parses cargo descriptions, CCU IDs, weights, DG classes, dimensions, and platform assignments automatically. Imported items appear in a queue for selective placement.

⚠ DG Dangerous Goods

20 IMDG DG classes with a full segregation matrix. The system automatically checks every cargo pair for DG violations. Conflicts show a pulsing red outline and a modal warning. Each conflict can be individually acknowledged.

📄 PDF Export

Generates a professional A4 landscape PDF with header, voyage info, stats, location cards, DG summary, full deck image, bay labels, voyage notes, and footer. Opens native Save As dialog — user chooses destination.

📊 Excel Export

Two-sheet Excel workbook: Cargo Manifest (all items with CCU, dimensions, weight, status, platform, DG) and Summary (totals, per-location breakdown, voyage notes). Native Save As dialog.

↶ Undo / Redo

50-step undo history. Every cargo mutation (move, add, delete, edit, rotate, duplicate) is tracked. Ctrl+Z to undo, Ctrl+Y or Ctrl+Shift+Z to redo. UI buttons also available in the header.

💾 Save / Open Project

Save the complete deck state as a .json project file. Open previously saved projects. Ctrl+S saves to current file (or opens Save As if new). Autosave to localStorage every 15 seconds as crash protection.

🔍 Zoom Controls

Zoom 30%–200% with smooth transitions. Ctrl+Scroll for wheel zoom. Number keys 1–5 for preset levels. Fit-to-screen button. Pinch-to-zoom on touch devices.

🌙 Dark / Light Theme

Full dark mode with warm graphite surfaces. Toggle via the theme pill in the header. All UI elements, cargo blocks, modals, and export dialogs are dark-mode aware.

📋 Workflow Guide

A typical voyage planning workflow from start to PDF export:

1

Set Voyage Details

Enter the voyage number and select the date in the header. Add voyage notes if needed.

2

Activate Platforms

Click the ⊕ button in the location strip to add the platforms this voyage will visit. Each gets a unique colour.

3

Import ASCO File (Optional)

Upload the ASCO/iLMS Excel loadout file. Review imported items in the queue. Select items to place on deck.

4

Place Cargo

Drag cargo from the Library panel onto the deck, or click a library card then click the deck to place. Position, rotate (R), and resize as needed.

5

Assign Status & Platform

Click each cargo block to open the edit modal. Set the correct status (Load/BL/ROB/TR), platform, weight, and DG class.

6

Review DG Compliance

Check for DG segregation warnings (red pulsing outlines). Acknowledge or resolve each conflict.

7

Export PDF

File → Export PDF. Choose destination folder and filename in the Save As dialog. The PDF includes full deck image, stats, and manifest data.

8

Export Excel

File → Export Excel for the formal cargo manifest spreadsheet with per-location breakdown.

9

Save Project

Ctrl+S to save the project file for future reference or continuation.

💡 Tips & Best Practices

★ Use Smart Grid Snap
Keep Smart Grid Snap enabled (default ON). It aligns cargo to bay lines and neighbours automatically, keeping the deck plan clean and realistic.
★ Zoom to 100% Before Export
For the best PDF quality, set zoom to 100% before exporting. The deck image is captured at the current view.
★ Use Platform Highlight
Enable "Highlight by Platform" in Smart Tools, then click a platform card to isolate its cargo visually. Great for verifying platform-specific loads.
★ Save Early, Save Often
Autosave protects against crashes (every 15s to localStorage), but Ctrl+S creates a proper named project file you can share or archive.
★ Right-Click Context Menu
Right-click any cargo block for quick access to Edit, Rotate, Duplicate, and Delete actions.
★ Keyboard Shortcuts Speed
Learn E/R/D/Del for cargo editing — much faster than using the mouse buttons. Arrow keys for precision positioning (Shift+Arrow = 1 metre).
★ DG Acknowledgement
DG warnings are per-cargo-pair, not per-class. Acknowledging a warning for two specific containers won't suppress future warnings for new pairs of the same DG class.
