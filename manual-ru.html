⚓ Обзор

SPICA TIDE — профессиональное приложение для планирования палубного груза, разработанное для судна снабжения платформ SPICA TIDE, работающего в Северном море под управлением NEO Energy. Приложение предоставляет интерактивный план палубы с перетаскиванием, визуализацией груза в реальном времени, цветовой маркировкой платформ, проверками безопасности DG и полным экспортом манифестов в PDF/Excel.

Приложение заменяет бумажные и табличные грузовые планы точным масштабируемым цифровым инструментом, которым офицеры могут пользоваться за рабочим столом или на планшете на мостике.

🚢 12 грузовых бэёв
Полная палуба от бэя 12 (корма) до бэя 1 (нос), каждый индивидуально размечен по реальным размерам судна.
⚖ 7 платформ
Bleo Holm, Claymore CAP/CPP/WOPS/Drill, Piper, Saltire — каждая с уникальной цветовой маркировкой и отслеживанием груза.
⚠ Безопасность DG
Полная матрица сегрегации IMDG с 20 классами DG. Автоматическое попарное обнаружение конфликтов с визуальными предупреждениями.
📄 Готово к экспорту
Профессиональные PDF-планы палубы и Excel-манифесты экспортируются напрямую на диск через нативные диалоги сохранения.

⚙ Основные функции

👁 Размещение перетаскиванием

Выберите груз из панели библиотеки и перетащите его прямо на палубу. Отпустите для размещения в точной позиции. Грузовые блоки можно перемещать перетаскиванием, поворачивать на 90° клавишей R, дублировать клавишей D или удалять клавишей Del.

📚 Панель библиотеки грузов

Нижняя панель содержит 60+ предустановленных типов грузов в 5 категориях: Контейнер, Модуль, Танк, Корзина, Скип. Каждый пресет включает реальные размеры в метрах и вес по умолчанию. Можно добавлять и сохранять пользовательские типы.

🎨 Цветовая маркировка платформ

Каждой платформе назначен уникальный цвет. Грузовые блоки окрашиваются по назначенной платформе, что позволяет легко визуально определить, какой груз куда направляется. Цвета можно менять вручную, нажав на цветную точку на карточке локации.

📌 Статусы грузов

Четыре статуса груза определяют цветовую тонировку и группировку в манифесте:

■ Load
Груз, загружаемый на судно для доставки
■ Backload
Груз, возвращаемый с платформы на берег
■ ROB
Остаётся на борту с предыдущего рейса
■ Transfer
Груз, передаваемый между платформами (пунктирная граница)

📄 Импорт ASCO / iLMS Excel

Загрузите Excel-файл ASCO (.xlsx), и приложение автоматически разберёт описания грузов, CCU ID, вес, классы DG, размеры и назначения платформ. Импортированные элементы появляются в очереди для выборочного размещения.

⚠ Опасные грузы (DG)

20 классов DG IMDG с полной матрицей сегрегации. Система автоматически проверяет каждую пару грузов на нарушения DG. Конфликты отображаются пульсирующей красной обводкой и модальным предупреждением. Каждый конфликт можно подтвердить индивидуально.

📄 Экспорт PDF

Генерирует профессиональный PDF формата A4 альбомной ориентации с заголовком, информацией о рейсе, статистикой, карточками локаций, сводкой DG, полным изображением палубы, номерами бэёв, заметками и нижним колонтитулом. Открывается нативный диалог «Сохранить как».

📊 Экспорт Excel

Книга Excel с двумя листами: Манифест грузов (все элементы с CCU, размерами, весом, статусом, платформой, DG) и Сводка (итоги, разбивка по локациям, заметки рейса). Нативный диалог сохранения.

↶ Отмена / Повтор

История отмены на 50 шагов. Каждое изменение груза (перемещение, добавление, удаление, редактирование, поворот, дублирование) отслеживается. Ctrl+Z для отмены, Ctrl+Y или Ctrl+Shift+Z для повтора. Кнопки также доступны в заголовке.

💾 Сохранение / Открытие проекта

Сохраните полное состояние палубы как .json файл проекта. Откройте ранее сохранённые проекты. Ctrl+S сохраняет в текущий файл (или открывает «Сохранить как» для нового). Автосохранение в localStorage каждые 15 секунд для защиты от сбоев.

🔍 Управление масштабом

Масштабирование 30%–200% с плавными переходами. Ctrl+Scroll для масштабирования колёсиком. Клавиши 1–5 для предустановок. Кнопка «По размеру экрана». Щипковое масштабирование на сенсорных устройствах.

🌙 Тёмная / Светлая тема

Полная тёмная тема с тёплыми графитовыми поверхностями. Переключение через переключатель темы в заголовке. Все элементы интерфейса, грузовые блоки, модальные окна и диалоги экспорта поддерживают тёмную тему.

📋 Рабочий процесс

Типичный рабочий процесс планирования рейса от начала до экспорта PDF:

1

Задайте данные рейса

Введите номер рейса и выберите дату в заголовке. При необходимости добавьте заметки рейса.

2

Активируйте платформы

Нажмите кнопку ⊕ в панели локаций, чтобы добавить платформы, которые посетит этот рейс. Каждой присваивается уникальный цвет.

3

Импорт файла ASCO (необязательно)

Загрузите Excel-файл ASCO/iLMS. Просмотрите импортированные элементы в очереди. Выберите элементы для размещения на палубе.

4

Разместите грузы

Перетащите груз из панели библиотеки на палубу или нажмите карточку библиотеки, затем нажмите на палубу для размещения. Позиционируйте, поворачивайте (R) и изменяйте размер по необходимости.

5

Назначьте статус и платформу

Нажмите на каждый грузовой блок, чтобы открыть модальное окно редактирования. Установите правильный статус (Погрузка/Выгрузка/ROB/Трансфер), платформу, вес и класс DG.

6

Проверьте соответствие DG

Проверьте предупреждения о сегрегации DG (красные пульсирующие контуры). Подтвердите или устраните каждый конфликт.

7

Экспорт PDF

Файл → Экспорт PDF. Выберите папку назначения и имя файла в диалоге «Сохранить как». PDF включает полное изображение палубы, статистику и данные манифеста.

8

Экспорт Excel

Файл → Экспорт Excel для официальной таблицы манифеста грузов с разбивкой по локациям.

9

Сохранить проект

Ctrl+S для сохранения файла проекта для будущего использования или продолжения.

💡 Tips & Best Practices

★ Используйте Smart Grid Snap
Оставьте Smart Grid Snap включённым (по умолчанию ВКЛ). Он автоматически выравнивает грузы по линиям бэёв и соседям, сохраняя план палубы чистым и реалистичным.
★ Масштаб 100% перед экспортом
Для лучшего качества PDF установите масштаб 100% перед экспортом. Изображение палубы захватывается в текущем виде.
★ Используйте подсветку платформ
Включите «Подсветку по платформе» в Умных инструментах, затем нажмите карточку платформы для визуальной изоляции её грузов. Отлично подходит для проверки грузов конкретной платформы.
★ Сохраняйте рано, сохраняйте часто
Автосохранение защищает от сбоев (каждые 15 сек в localStorage), но Ctrl+S создаёт полноценный именованный файл проекта, который можно передать или архивировать.
★ Контекстное меню (правый клик)
Нажмите правой кнопкой на любой грузовой блок для быстрого доступа к редактированию, повороту, дублированию и удалению.
★ Скорость горячих клавиш
Выучите E/R/D/Del для редактирования грузов — намного быстрее, чем использование кнопок мыши. Стрелки для точного позиционирования (Shift+Стрелка = 1 метр).
★ Подтверждение DG
Предупреждения DG привязаны к конкретной паре грузов, а не к классу. Подтверждение предупреждения для двух конкретных контейнеров не подавит будущие предупреждения для новых пар того же класса DG.
